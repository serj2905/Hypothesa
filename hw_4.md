# Домашнее задание №4 — Данные

> Все цифры посчитаны по реальному `DHA_hackathon/data/.../results.csv` (n = 25). Воспроизведение: `python -m eval.eda`.

## 1. Что за данные

**Первичные данные**, которые порождает сама система: Telegram-бот ведёт интервью → диалог парсится по вопросам → LLM суммаризирует ответы → всё сводится в `results.csv`.

| Параметр | Значение |
|---|---|
| Записей | 25 |
| Столбцов | 8 — 4 пары `user_ans_*` (сырой ответ) / `summed_ans_*` (LLM-суммаризация) |
| Язык | Русский |
| Разметка | Меток нет; «псевдоразметка» создаётся LLM |

| Вопрос | Колонки | Ожидаемый тип |
|---|---|---|
| Возраст | `*_1` | число |
| Город | `*_2` | топоним |
| Что не нравится в Сбере | `*_3` | список проблем |
| Что нравится в Сбере | `*_4` | список плюсов |

**Целевая задача — не классификация.** Итог пайплайна — кластеризация открытых ответов (BERTopic) + рейтинг тем. Рабочий корпус — открытые вопросы 3 и 4; возраст/город — демографический контекст.

## 2. EDA (ключевые выводы)

**Пропуски** растут к концу анкеты: вопрос 1–2 — 4 %, вопрос 3 — 8 %, **вопрос 4 — 20 %** (+1 полностью пустая строка). Позитивный корпус меньше негативного.

**Демография:** возраст 18–60, медиана **26** (молодая выборка, как канал Telegram); города — **СПб 54 %**, остальные единично. → сегментация по демографии невозможна, только описательный контекст.

**Длина открытых ответов:** медиана **8 слов**, разброс 1–105 (скошено). ~12–15 % ответов — заглушки («Все», «Ничего»). → перед кластеризацией нужен фильтр пустых.

**Формат суммаризации (главная проблема):** промпт просит Python-список, но он получен лишь в **3/25** (вопрос 3) и **0/25** (вопрос 4); 11 «суммаризаций» — это вообще отказы LLM («К сожалению, вы не предоставили…»). → regex-парсер теряет данные, нужен structured output.

**Артефакты в данных:** галлюцинации (банковские проблемы → «синтаксис Python»; «перевод по телефону» → «профессиональные переводчики»), сдвиг колонок (возраст = «Хорошо»), захват фразы «До свидания» как ответа. → нужны валидация схемы + фильтр faithfulness.

> Важно: галлюцинации порождала устаревшая модель GigaChat от Сбера, использованная в прототипе. В улучшенной версии она заменяется на более современную LLM, что вместе со structured output и Judge-2 должно резко снизить долю искажений.

## 3. Качество разметки и как улучшить

«Разметка» = качество суммаризации + стабильность тем + рейтинг (всё от LLM, не от человека). Проблемы и меры:

| Проблема | Замер | Решение |
|---|---|---|
| Нарушение формата | список в 3/25 и 0/25 | structured output (JSON-схема) |
| Отказы/галлюцинации | 11 отказов + ≥4 галлюц. | Judge-2 faithfulness + детектор отказов |
| Пустые ответы | ~6 заглушек | фильтр по длине + словарю |
| Нестабильные темы | LLM-группировка не воспроизводима | BERTopic с фикс. `random_state` |

## 4. Формирование выборки и валидация

**Выборка:** очистка (убрать NaN/отказы/галлюцинации/заглушки) → нормализация → explode списков в отдельные документы → эмбеддинги (`paraphrase-multilingual-MiniLM-L12-v2`) → BERTopic. Ожидаемо ~40–50 документов.

**Валидация (test-set нет):**
- автометрики кластеров: Coherence Cᵥ ≥ 0.50, Silhouette ≥ 0.25, Noise ≤ 20 %, Coverage топ-5 ≥ 60 %;
- 4 LLM-судьи (диалог, faithfulness, категории, рейтинг);
- скользящая проверка рейтинга по мере роста выборки.

⚠️ При n = 25 (≈15 информативных ответов) выборка пилотная — для значимой кластеризации нужно **50–100 валидных интервью**.
